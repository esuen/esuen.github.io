What it does

Deep Dive is a podcast production pipeline that researches any topic, verifies every factual claim against sources, and converts the research into a voiced podcast with per-line emotional delivery directions. The same verified research can be repurposed for video (with frame-accurate synchronized visuals) or a blog post when needed.

The research layer is format-agnostic—the same verified, confidence-tagged research synthesis can drive any output format. Research once, output to whichever platform fits the topic.

Under the hood

Dual orchestration with quality gates

A pipeline.py CLI of composable steps drives episode-by-episode production with manual confirmation before expensive operations. The same steps are wired into a Dagster DAG (16 software-defined assets, 8 quality checks, web UI) for batch reruns and observability. CLI for production control, DAG for introspection. Quality checks gate each stage either way: Whisper hallucination detection after voicing, adjacent-block contamination checks, content timing verification after video assembly.

Multi-agent research synthesis

Each episode kicks off four parallel sub-agents (~3 hours wall time): technical/benchmark, strategic/business, alignment/safety (or domain-specific), and labor/political. Each produces a 3,000–5,000-word brief with confidence tags ([VERIFIED], [SINGLE SOURCE], [ANALYTIC]) and direct WebFetch citations. A synthesis pass compresses 12,000–17,000 words of backing into a 6,000-word unified brief with cold-open candidates, section structure, contradictions and hedges, and 5 testable predictions.

News-scout pipeline

Daily 6 AM PDT launchd job runs Claude headless across ~50 sources. Scores each candidate 0–70 across 7 dimensions: depth, numbers, underexplored, stakes, verifiability, timeliness, fit. Dedup window prevents re-rescoring same story across consecutive days. Topics in PRIORITIZE tier (≥50) feed episode selection. Discovery rate: ~3–5 strong candidates per day, ~1 episode-worthy story per week.

Two-pass review + tic audit before voicing

Every script goes through two parallel sub-agents: style/balance/pacing review (AI tics, em-dash density, transitions) and fact-check against research briefs (every numeric claim, quote attribution, date). Fact-check has caught 3–7 errors per episode. Then tic_audit.py runs a quantitative validator: weighted-tag percentage, em-dashes per 1000 words, section-opener forms, trailing 1–4 word fragments, compound word years, "Confidence high/medium" spoken stamps, "Not X — Y" patterns, stacked stats, short-line floor, slide-candidate density. Each upstream gate is cheaper than the operation it protects.

Multi-platform output from one voicing pass

One full script (~20 min, Spotify) is voiced once (~$1.50). An LLM cuts it for YouTube by identifying the weakest sections—audio spliced at block boundaries, no re-voicing. Blog posts and short clips derive from the same source. Marginal cost of new platform output: ~$0.05 LLM.

Narration-synced animated visuals (when video required)

For episodes that ship video, Manim animated visuals are synced to the exact word being spoken via Whisper word-level timestamps. Animations trigger on specific narration words—a map zooms out when the host says “geographically,” ship icons appear on “ships,” a stat card flashes on the key number. Script-line-aware matching prevents common-word false triggers. Templates: GridDashboard, PriceLine, ScaleComparison, MapScene (with matplotlib backgrounds), TypewriterQuote. The Strait of Hormuz episode used 24 Manim visuals; most episodes ship audio-first.

TTS hallucination + dropout prevention

Two-tier detection: duration check (free), then Whisper transcription verification (re-voice up to 3 times). A pre-voice transform converts period-separated lists to comma format—preventing TTS from hallucinating repeated words. Trailing 1–4 word fragments after long sentences (recurring TTS dropout) caught by the tic-audit validator pre-voice. Adjacent-block contamination check catches words leaking across blocks. Voice consistency profiling via Praat (F0, formants, HNR) detects and fixes gender/timbre switches.

Notion-integrated production system

Four databases under "Deep Dive" workspace, integrated via MCP: Episodes (Status, Topic Tier, Duration, Hero Stat, Spotify URL), Predictions (40+ rows with check-in dates, powering callbacks in future episodes), Topic Ideas (researched backlog with demand-check notes), Production Tasks. The Predictions DB is a compounding asset—6 months of predictions become the 6-month-retrospective episode.

Decisions and tradeoffs

Single host over two hosts. Tested every OpenAI and ElevenLabs voice. All two-voice combinations sounded detectably AI. A single male voice with per-line emotional directions sounds more natural. The script uses rhetorical questions instead of a second host.

Never overwrite inputs. Each pipeline step outputs to a new file. We lost a 30-minute generated script early on by running normalize and merge in-place. Now enforced by the tools refusing identical input/output paths.

Single-pass audio chain over multi-encode. Each lossy MP3 re-encoding compounds artifacts. EP25 went through three re-encodings (per-block 128k → custom rebuild → finalize at 192k 48kHz) and audibly degraded vs episodes that went through two. Standardized on a single concatenation pass at 192k 48kHz, with explicit ffmpeg -b:a 192k parameter (pydub's bitrate= kwarg silently defaults to 32 kbps in some configurations).

Verification at every layer. Research verification, script-vs-research fact-checking, number normalization tests, pronunciation fixing, voice consistency detection, visual spelling verification. The project has more verification code than generation code.

Learnings

AI doesn’t enforce its own rules. Instructions say “never guess.” It guesses anyway—wrong timestamps, wrong slide content, wrong diagram labels. Adding more instructions didn’t help. The fix: encode every rule as a test or pipeline gate. 8 automated quality checks block bad output before it ships. Instructions are suggestions. Code is enforcement.

Verification matters more than generation. The project has more verification code than generation code. Research sourcing, claim checking, voice consistency detection, visual timing verification, script anti-pattern scanning, adjacent-block contamination checks. AI-generated content without a trust layer is automated misinformation.

The pipeline is the product. The episodes are outputs, but the real value is the system: research any topic, verify claims, produce platform-optimized content automatically. The marginal cost of a new episode is ~$1.50 TTS + ~$0.05 LLM.

What’s next

Short-form clips pipeline. Each episode has 5–10 60–90s segments with strong hooks. Automating vertical-crop with caption overlay for X, TikTok, and Reels. The single biggest discoverability lever for podcast growth in 2026.

Predictions check-in workflow. 40+ predictions logged in Notion DB with check-in dates. Every landed prediction is a “we called this 90 days ago” moment that drives back-catalog discovery. Building automated check-in + retrospective-episode generation.
