What it does

Deep Dive is a content production pipeline that researches any topic, verifies every factual claim against sources, and converts the research into multiple output formats: a voiced podcast with per-line emotional delivery directions, a blog post with comparison tables and diagrams, and a video with frame-accurate synchronized visuals.

The research layer is format-agnostic—the same verified, confidence-tagged research produces a podcast, a blog post, or a presentation. Research once, output to multiple formats.

Under the hood

Confidence-aware claims

Research tags every claim as [VERIFIED] (2+ sources), [SINGLE SOURCE], or [CLAUDE KNOWLEDGE]. The script automatically hedges low-confidence claims. Verification checks URLs are reachable and flags unsourced numbers. Three-layer verification: research quality, script-vs-research accuracy, and structural quality.

Per-line TTS delivery directions

Every line includes a delivery instruction: “stunned disbelief, emphasizing SEED” or “rattling off data with confidence.” Generated inline with the script so the writer knows the emotional intent. Baseline voice instruction prevents gender switching.

Automated diagram generation

Scans the script for technical concepts, generates image prompts, calls OpenAI’s image API, OCR-verifies for spelling errors and stray text, and regenerates on failure (up to 3 attempts). Also detects any visual gap >2 minutes and suggests diagrams for it.

Slide-aware line merging

Consecutive lines merge into single TTS calls for natural flow—but lines tagged as slide candidates stay separate so progressive bullet slides sync precisely to audio. Merging reduces TTS calls by ~50% while preserving visual timing.

Context-aware number normalization

Converts spelled-out numbers to digits while respecting context: “one hundred twenty-six” → “126” but keeps “this one is” as a word. Handles compound hundreds/thousands. 22 test cases prevent regression of a bug that shipped wrong numbers in two episodes before being caught.

Frame-accurate video assembly + verification

Video built frame-by-frame using PIL—no segment concatenation (which causes timing drift). After assembly, automated verification extracts frames at the start, middle, and end of every visual segment and compares against the expected image using pixel similarity. Episode 3: 60/60 segments verified correct.

Voice consistency detection

OpenAI TTS switches voice on short utterances because each API call is stateless. Solved with three fixes: pinned model snapshots for deterministic behavior, concise voice instructions (verbose ones cause drift—confirmed OpenAI bug), and Praat-based acoustic analysis that builds a per-episode voice profile (F0, formants, HNR) and flags blocks deviating on multiple dimensions. Flagged blocks are automatically re-voiced and spliced back at exact timestamps.

Content timing verification

Slides must show when their content is spoken, not just when their block starts. Each slide carries metadata mapping it to keywords in the audio. After video assembly, a verifier checks every slide’s timestamp against the durations file—confirming the audio at that moment matches the slide content. Catches the disclaimer offset bug that shifted all visuals by 11.5 seconds.

Spoken-to-visual text conversion

TTS scripts spell out numbers (“99 point 9 percent”) but slides need the visual form (“99.9%”). A conversion layer handles percentages, currency ($12.5B), data units (2.7 TB), time units (200ms), bit rates (80 Gbps), multipliers (4x), ordinals (#1), and degrees (45°). For lines where spoken text is an analogy (“Half a nanosecond to read it”), a slide_display field provides the visual override (“L1 Cache — 0.5ns”). 33 tests cover the conversion rules.

DAG-based pipeline orchestration

Migrated from a sequential CLI to Dagster software-defined assets. Each pipeline step is an asset with declared dependencies—Dagster resolves execution order, tracks which outputs are stale when inputs change, and provides a web UI for pipeline observability. Quality checks gate each stage: Whisper hallucination detection after voicing, adjacent-block contamination check, silence padding detection, content timing verification after video assembly, and OCR diagram accuracy.

TTS hallucination prevention

Two-tier detection: duration check (free—if a block with <15 words generates >1.5x expected duration, flag it) then Whisper verification (transcribe flagged blocks, compare word count, re-voice up to 3 times). A pre-voice transform converts period-separated lists (“Servers. Databases. Caches.”) to comma format with “and” before the last item—preventing TTS from pattern-completing and hallucinating repeated words. Adjacent-block contamination check verifies the next block doesn’t start with leaked words from the current block.

Multi-format output

Same verified research produces multiple formats. Podcast scripts convert to blog posts with comparison tables and embedded diagrams. Social media clips slice from the actual episode audio using duration timestamps. Auto-generated descriptions with exact timestamps for YouTube and Spotify.

Decisions and tradeoffs

Single host over two hosts. Tested every OpenAI and ElevenLabs voice. All two-voice combinations sounded detectably AI. A single male voice with per-line emotional directions sounds more natural. The script uses rhetorical questions instead of a second host.

Never overwrite inputs. Each pipeline step outputs to a new file. We lost a 30-minute generated script early on by running normalize and merge in-place. Now enforced by the tools refusing identical input/output paths.

Frame-by-frame video over concatenation. Early versions concatenated ffmpeg segments, causing audible stuttering and timing drift. Now the video is built frame-by-frame—one continuous audio track with images placed at exact timestamps.

Verification at every layer. Research verification, script-vs-research fact-checking, number normalization tests, pronunciation fixing, voice consistency detection, visual spelling verification. The project has more verification code than generation code.

Learnings

Constraints drive better design. Single host, immutable files, confidence tags—each constraint initially felt like a limitation but produced a better system.

Verification matters more than generation. The project has more verification code (research sourcing, claim checking, pronunciation fixing, voice consistency detection, video timing verification) than generation code. AI-generated content without a trust layer is just automated misinformation.

Automate the tedious, review the creative. Image generation, pronunciation, number normalization, volume leveling—automate. Script quality, diagram accuracy, content depth—human review. The pipeline removes you from mechanical tasks but keeps you in for judgment calls.

TTS consistency is an unsolved problem. OpenAI’s TTS is nondeterministic—no seed parameter, no temperature control. Short utterances like “Stop.” or “Passes.” give the model too little context to anchor voice identity, causing gender or timbre switches. Zero-crossing rate pitch detection misses subtle changes. Switching to Praat-based acoustic profiling (fundamental frequency, formants, harmonics-to-noise ratio) catches more, but the real fix was upstream: pin the model snapshot and shorten the voice instruction. Verbose instructions cause the model to drift—a confirmed OpenAI bug.

Spoken text and visual text are different languages. TTS needs “99 point 9 percent” but a slide needs “99.9%.” Analogies like “Half a nanosecond to read it” work in audio but mean nothing on screen without the label “L1 Cache.” The fix was two layers: an automatic converter for mechanical formatting (percentages, units, currency) and a slide_display field on every data line for cases where the spoken form is a narrative the viewer can’t parse visually.

Timing bugs compound across layers. The disclaimer prepended to the audio shifted all visual timestamps by 11.5 seconds. The durations file used raw timestamps, but the video used final timestamps. Progressive slides used hardcoded 4-second intervals instead of slide_candidate block times. Each layer assumed the previous layer’s coordinates. The fix: every timestamp flows from one source of truth (the durations file), with the disclaimer offset applied once at video assembly.

AI doesn’t enforce its own rules. Claude’s global settings say “never guess, never assume.” It guessed anyway—wrong slide timestamps, wrong bullet content, wrong diagram architecture. Adding more instructions didn’t help. The fix was encoding rules into pipeline code: automated tests that block bad output, verification steps that compare against source-of-truth data, and pipeline pauses that force human review. Instructions are suggestions. Code is enforcement.

The pipeline is the product. The podcast episodes are outputs, but the real value is the system: a tool that can research any topic, verify its claims, and convert it to consumable content across multiple formats.

What’s next

Parallel asset execution. Dagster supports parallelizing independent assets—voice consistency check, visual generation, and social clips can run simultaneously instead of sequentially.

More output formats. Presentation slides, newsletter, short-form video clips. Same verified research, more surfaces.
